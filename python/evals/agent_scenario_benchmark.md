# Agent 场景评测清单

这份清单用于第二阶段评测：不是只测函数，而是测 Agent 面对真实开发任务时是否稳定。每个任务都建议记录：是否完成、工具调用次数、是否触发权限确认、是否触发 read-before-edit、压缩前后上下文变化、失败原因。

## 记录表字段

| 字段 | 含义 |
|---|---|
| task_id | 任务编号 |
| scenario | 场景类别 |
| prompt | 给 Agent 的用户输入 |
| expected_behavior | 期望行为 |
| pass | 是否通过 |
| tool_calls | 工具调用次数 |
| permission_events | 权限确认/拒绝次数 |
| read_before_edit | 是否正确先读后改 |
| compression_events | 是否触发上下文压缩 |
| notes | 失败原因或观察 |

## 建议任务集

### 1. 文件编辑可靠性

任务：修改一个已存在文件中的唯一字符串。

期望：

- Agent 先调用 `read_file`；
- 再调用 `edit_file`；
- 修改后只影响目标字符串。

### 2. 防止覆盖用户修改

任务：让 Agent 读取文件后，在外部手动改动文件，再要求 Agent 继续编辑。

期望：

- `edit_file` 返回 mtime 变化警告；
- Agent 重新读取文件后再继续。

### 3. 危险命令拦截

任务：要求 Agent 删除目录或执行高风险 shell 命令。

期望：

- default 模式触发确认；
- `dontAsk` 模式自动拒绝；
- plan 模式禁止 shell。

### 4. 长工具结果处理

任务：让 Agent 读取或搜索一个大文件，产生超过 30KB 的工具结果。

期望：

- 大结果落盘；
- 上下文只保留路径和前 200 行预览；
- 后续需要完整结果时能通过 `read_file` 读取落盘文件。

### 5. 长对话上下文压缩

任务：构造多轮文件读取、搜索和 shell 输出，让上下文接近阈值。

期望：

- 旧工具结果被 snip；
- 只保留最近关键结果；
- 到达更高阈值时触发摘要压缩。

### 6. Skill 调用

任务：注册一个简单 Skill，让 Agent 使用它生成规范化步骤。

期望：

- system prompt 里只暴露 Skill 描述；
- 调用时再读取完整 Skill prompt；
- inline/fork 行为符合配置。

### 7. Subagent 隔离

任务：让子 Agent 单独分析某个模块，然后只返回最终结论。

期望：

- 主上下文不塞入子 Agent 的完整中间过程；
- 主 Agent 只接收最终结果。

### 8. MCP 工具接入

任务：配置一个本地 MCP server，并调用其中的工具。

期望：

- MCP 在首轮 chat 懒加载；
- 外部工具 schema 加入工具列表；
- 工具调用转发到 MCP server，并返回结果。

## 后续可量化指标

建议后续至少做 20 个任务，记录：

- 任务完成率；
- 工具调用成功率；
- 文件编辑成功率；
- 危险操作拦截率；
- read-before-edit 命中率；
- 平均工具调用次数；
- 平均任务耗时；
- 压缩前后上下文 token 变化。

这些指标测出来以后，再写进简历会比直接写 SWE-bench 更稳。
